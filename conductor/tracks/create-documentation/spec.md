# Specification: Document Commit History

## Objective

Create a comprehensive and beginner-friendly documentation of the project's commit history. The documentation should explain the changes introduced in each commit made by the author "wkmanire", providing clear justifications for a novice programmer.

## Deliverables

1.  A new `docs` directory at the root of the project.
2.  A markdown file named `changes.md` inside the `docs` directory.
3.  The `changes.md` file should contain a chronological a summary of each commit authored by "wkmanire".
4.  Each commit summary should include:
    *   The commit hash.
    *   The commit message.
    *   A detailed, step-by-step explanation of the changes.
    *   The justification for the changes, explained in simple terms.
